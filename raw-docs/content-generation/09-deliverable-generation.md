# Step 09 — Deliverable Generation & Download (Phase 6)

Package all generated content into the final deliverable set: per-page markdown files, a Word document, llms.txt, llms-full.txt, and robots.txt. Bundled as a downloadable zip.

---

## What We're Building

A deliverable assembly pipeline that runs once when Phase 5 completes. Produces a zip file stored in Supabase Storage, downloadable from the Phase 6 card.

## Word Document

Use the existing `docx` skill pattern (`lib/pdf/` style). One Word document containing all pages in order, organized by section matching the sitemap hierarchy.

Structure:
- Cover page: Firm name, date, "Website Content Package — Prepared by CountingFive"
- Table of contents (auto-generated by Word styles)
- Each page as a chapter: H1 as Heading 1, H2s as Heading 2, body as Normal style
- Metadata appendix at end: one table row per page with meta title, meta description, target keyword, URL slug

Use `docx` npm package (already available via the docx skill pattern):

```typescript
import { Document, Packer, Paragraph, HeadingLevel, TableCell, TableRow } from 'docx'
```

Store at: `content-packages/{sessionId}/content.docx`

## llms-full.txt (llmstxt.org spec)

Same header and structure as llms.txt, but each page entry expands to include the full `content_markdown` for that page. Separated by `---` between pages.

This file can be large (20–40k words for a full site). Store in Supabase Storage alongside the zip.

## Zip Assembly

Use the `archiver` npm package:

```bash
npm install archiver
npm install --save-dev @types/archiver
```

Assembly route: `POST /api/content-jobs/[id]/package`

1. Generate all markdown files in memory
2. Generate Word doc via `Packer.toBuffer()`
3. Generate llms.txt, llms-full.txt, robots.txt as strings
4. Zip all files using `archiver`
5. Upload zip to Supabase Storage: `content-packages/{sessionId}/content-package.zip`
6. Save storage path to `content_jobs` (add `package_url` column in migration)
7. Advance `content_jobs.phase` to `6`

## Phase 6 Card UI

Simple download card. Shows:
- ✓ Content generation complete
- File count summary: "23 pages · 1 Word document · 3 support files"
- "Download Package" button → hits a signed URL endpoint for the zip
- Individual download links for llms.txt, llms-full.txt, robots.txt separately

Signed URL endpoint: `GET /api/content-jobs/[id]/download` — generates a 60-minute Supabase Storage signed URL for the zip.

## Test Process

1. Complete Phase 5 — confirm Phase 6 card appears with "Download Package" button
2. Click "Download Package" — confirm zip downloads successfully
3. Open zip — confirm expected folder structure
4. Open one markdown file — confirm frontmatter, copy, and metadata appendix are all present
5. Open Word document — confirm table of contents, all pages present, metadata appendix at end
6. Check llms.txt — validate against llmstxt.org spec (header, blockquote description, grouped links)
7. Check llms-full.txt — confirm full copy is present for each page
8. Check robots.txt — confirm all AI crawlers are explicitly allowed
9. Run `npx tsc --noEmit` — zero errors
